# Prompt نهائي وصارم — إرساله كاملاً كما هو للوكيل (Claude Code أو أي وكيل برمجي)

---

## دورك

أنت مسؤول عن تحويل جذري كامل لمشروع `SmartBot` (لوحة إدارة صفحة فيسبوك) إلى مستوى لوحة تحكم عالمية احترافية. نفّذ هذا العمل وكأنك فريق من 4 مختصين يراجع كل منهم عمل الآخر قبل الاعتماد النهائي — إن كانت بيئتك تدعم Subagents فعلياً فاستخدمها بهذا التقسيم، وإن لم تدعم فتبنَّ كل دور بالتسلسل بنفس الصرامة دون تخطي أي مراجعة:

1. **مهندس البنية (Architecture Engineer):** يبني الكود المصدري الحقيقي من الصفر.
2. **قائد نظام التصميم (Design Systems Lead):** يقفل هوية بصرية واحدة ولا يسمح بتغييرها لاحقاً.
3. **مهندس الواجهة (Frontend Builder):** يبني كل صفحة على نظام التصميم المُقفل فقط.
4. **مدقق الجودة (QA Auditor):** يرفض أي تسليم لا يجتاز قائمة التحقق كاملة، ويعيده للمرحلة المسؤولة.

## الحقيقة التي يجب ألا تتكررها (اقرأ هذا جيداً)

فحصت تاريخ Git الكامل لهذا المشروع: **كل تاريخه، 40 commit، يمتد بين الساعة 16:01 و21:54 من نفس اليوم**. خلال هذه الساعات الست تم استبدال الهوية البصرية بالكامل 4 مرات متتالية (React أولي → "rose-dark" → "Flip7 teal/coral/gold" → "SaaS neutral gray") — وفي كل مرة عبر رفع ناتج بناء (dist) جاهز مباشرة إلى `static/assets`، **دون أن يُرفع الكود المصدري الحقيقي (`src/`, `package.json`) لأي commit إطلاقاً**، ولو مرة واحدة، طوال تاريخ المشروع.

**النتيجة:** لا يوجد اليوم أي كود مصدري قابل للتعديل. `App.jsx` و`main.jsx` الموجودان في الجذر يستوردان مكونات وملفات (`@/components/ui/sidebar`, `@/pages/dashboard`, `./index.css`، وغيرها) **غير موجودة إطلاقاً**.

**القاعدة الآمرة رقم 1:** ممنوع منعاً باتاً تكرار هذا النمط. كل تغيير تصميمي يجب أن يكون عبر **تعديل كود مصدري حقيقي يُرفع كاملاً لـ Git** (`src/`, `package.json`, كل ملف مكوّن) — وليس عبر توليد ناتج بناء مُصغّر ورفعه مباشرة بدون مصدره.

**القاعدة الآمرة رقم 2:** بمجرد إقفال نظام التصميم في المرحلة 1 (انظر أدناه)، **ممنوع تغيير أي لون أساسي أو خط أو مقياس مسافة** في بقية المراحل إلا إذا طلب المستخدم ذلك صراحة بالاسم. لا "تحسينات" تصميمية عشوائية أثناء بناء الصفحات — الالتزام بالتوكنز المُقفلة حرفياً.

## السياق التقني الحالي (لا تُعد بناءه، استخدمه كما هو)

الـ Backend (FastAPI) يعمل بالفعل ويوفر كل النقاط التالية — لا تُعد كتابتها، فقط اربط الواجهة بها:
`GET /api/stats` · `GET/POST/PUT/DELETE /api/rules[/{id}]` · `POST /api/rules/{id}/toggle` · `GET /api/replies?page&per_page` · `GET /api/posts` · `POST /api/publish` · `GET /api/bot/status` · `POST /api/bot/restart` · `GET /api/logs?limit`

**ناقص من الـ Backend ويجب إضافته في المرحلة 0:** مصادقة كاملة (`/api/login`, جلسة JWT في httpOnly cookie)، جدول `User` بأدوار (`admin` / `editor` / `viewer`)، وحماية كل نقطة حساسة بالتحقق من الدور فعلياً في الـ Backend نفسه — وليس فقط إخفاء الأزرار في الواجهة.

## نظام التصميم المُقفل (لا تناقشه، طبّقه حرفياً)

**المفهوم:** "غرفة التحكم" (Control Room) — الهوية مبنية على أن SmartBot يراقب صفحة فيسبوك حيّة ويرد تلقائياً 24 ساعة. العنصر المميز الوحيد: مؤشر **"نبض حي" (Live Pulse)** ثابت في الشريط العلوي (نقطة تتنفس + "البوت نشط الآن") — هذا هو العنصر الحركي الوحيد المسموح أن يلفت الانتباه؛ كل شيء آخر هادئ ومنضبط.

**الألوان (الوضع الداكن):**
```
--bg:        #0D1017   (Graphite Night)
--surface:   #161B24   (Slate Panel)
--border:    #262C3A   (Steel Line)
--text:      #EDEFF4   (Mist White)
--accent:    #FF5D3A   (Ember — أساسي/حي/فعّال)
--info:      #4C8DFF   (Signal Blue — بيانات/روابط)
```
**الألوان (الوضع الفاتح):**
```
--bg:        #F7F8FA
--surface:   #FFFFFF
--border:    #E4E7EC
--text:      #14171F
--accent:    #E14A22
--info:      #2F6FE0
```
ممنوع استخدام أي لون خارج هذه اللوحة (باستثناء أخضر/أحمر حياديين محددين فقط لحالات نجاح/خطأ النظام، منفصلين عن هوية العلامة). ممنوع الاعتماد الكلي على `oklch()` — استخدم Hex/HSL قياسي متوافق مع كل المتصفحات.

**الطباعة:** IBM Plex Sans Arabic لكل النصوص والعناوين (أوزان 400/500/600/700) — ممنوع `system-ui`. IBM Plex Mono لكل الأرقام (KPIs، السجلات، الطوابع الزمنية، المعرّفات).

**القياسات:**
```
Spacing (rem): 0.25 / 0.5 / 0.75 / 1 / 1.5 / 2 / 3 / 4 / 6
Radius:        sm=6px  md=10px  lg=14px
Shadow:        فقط عند hover/focus/dialogs — لا ظل ثابت على البطاقات
Motion:        سريع=120ms  عادي=200ms  بطيء=320ms — ease-out للدخول
Breakpoints:   375 / 768 / 1280 / 1600 px
اتجاه:          RTL كامل، الشريط الجانبي على اليمين
```

**الصفحات المطلوبة:** تسجيل الدخول · لوحة القيادة (KPIs + رسم مساحي + Donut + نشاط مباشر بشكل محادثة) · القواعد (جدول بحث/فرز + Side Sheet للإضافة/التعديل، ليس Dialog مركزي) · سجل الردود (بشكل بطاقات محادثة: تعليق فوق رد) · المنشورات (شبكة بطاقات + معاينة حية قبل النشر) · الإعدادات (تبويبات: عام/حالة البوت/السجلات/المستخدمون) · المستخدمون والصلاحيات (جديدة بالكامل).

## خطة التنفيذ المرحلية الإجبارية (لا تقفز مرحلة، ولا تُنفّذ كل شيء دفعة واحدة)

### المرحلة 0 — الأساس (حاسم، لا تتجاوزه)
ابنِ `package.json` + مجلد `src/` كامل (`components/`, `pages/`, `index.css`) بما يتوافق مع ما تستورده `App.jsx`/`main.jsx` الحاليان أو أعد كتابتهما بنفس البنية المعمارية (React + Vite + shadcn/ui + Tailwind + Framer Motion + TanStack Query + Sonner). أضف نظام المصادقة والأدوار في الـ Backend.
**بوابة العبور:** `npm install && npm run build` ينجح بدون أي خطأ. تسجيل الدخول يعمل فعلياً ويحمي كل الصفحات.

### المرحلة 1 — إقفال نظام التصميم
طبّق التوكنز أعلاه في ملف CSS مركزي واحد + `tailwind.config`. ابنِ صفحة **واحدة فقط** (لوحة القيادة) للتحقق البصري.
**بوابة العبور:** اعرض لقطة/وصف دقيق لصفحة لوحة القيادة قبل المتابعة — لا تكمل بقية الصفحات قبل التأكد أن الاتجاه صحيح.

### المرحلة 2 — بناء كل الصفحات
ابنِ باقي الصفحات (القواعد، الردود، المنشورات، الإعدادات، المستخدمون) باستخدام نفس التوكنز المُقفلة فقط، بلا أي انحراف تصميمي.
**بوابة العبور:** كل صفحة بلا أخطاء console وبلا overflow أفقي على 375px و1280px.

### المرحلة 3 — الصلاحيات والحالات والحركة
فعّل فحص الأدوار الفعلي (وليس الشكلي) على كل زر حذف/تعديل/نشر. أضف Skeletons، Empty States، Error States، Toasts، ومؤشر النبض الحي.
**بوابة العبور:** اختبر يدوياً بحساب `viewer` — تأكد أنه لا يرى أزرار الحذف/التعديل، وأن استدعاء الـ API مباشرة (curl) بحسابه يُرفض من الـ Backend أيضاً.

### المرحلة 4 — التنظيف والتحقق النهائي
احذف ملفات البناء المكررة في `static/assets`، احذف/أرشف `templates/index.html` القديم، أضف `.gitignore` مناسباً، واكتب اختبار Playwright يغطي: تسجيل دخول → تصفح كل صفحة → إضافة قاعدة → تعديلها → حذفها.
