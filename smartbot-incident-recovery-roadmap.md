### 1) الوكيل عمل `force-push` — كتب فوق تاريخ الـ git بالكامل

```

```

هذا معناه: مسح تاريخ الفرع الرئيسي وكتب فوقه تاريخ جديد. تتبعت نقطة
الانشقاق:

```

→ bde06d91   (نفس كوميت الصورة اللي كنا نتحدث عنها، بتاريخ 12 يوليو!)
```

**يعني الوكيل عمل بالضبط اللي حذّرت منه بكل برومبت أرسلته:** رجّع
المشروع كامل لكوميت 12 يوليو (بأدوات SPA/React Router القديمة)،
وبنى فوقه من الصفر بدل ما يبني فوق آخر نسخة (بدأ الساعة 17:00،
بعد آخر تحقق لي بـ 42 دقيقة بس).

**النتيجة: 521 ملف تغيّر، 24,494 سطر انمسح، 14,778 سطر جديد.** هذا
مو تعديل — هذا استبدال شبه كامل للمشروع.

## خط الاستعادة والإصلاح والتنظيم

### المرحلة 0 — إنقاذ فوري (اليوم، قبل أي شي ثاني)

1. **أوقف الوكيل عن أي عمل جديد فورًا.**
2. من جهازك (عندك صلاحية push)، شغّل:
   ```
   git fetch origin
   git branch recovery-9b10d782 9b10d782
   git push origin recovery-9b10d782
   ```
   لو `9b10d782` مو موجود عندك محليًا، خلي الوكيل يجيبه بهذا الأمر
   أولاً (لسا موجود بـ reflog الوكيل على الأغلب لأنه هو اللي عمل
   الـ reset):
   ```
   git reflog | grep 9b10d782
   ```
3. **لا تلمس main حتى تتأكد إن `recovery-9b10d782` انرفع بنجاح
   لـ GitHub وتقدر تشوفه بالمتصفح.** هذا يأمّن نسخة لا تنضاع
   بغض النظر عما يصير بعدين.

### المرحلة 1 — استعادة Vercel

1. ادخل Vercel dashboard تأكد هل مشروع `bot.smart-link.ly` محذوف
   فعليًا أو بس الدومين انفك. لو محذوف بالكامل، أنشئ مشروع جديد
   من فرع `recovery-9b10d782` واربط الدومين عليه.
2. تأكد `api.bot.smart-link.ly` (الباك-إند الأصلي) لسا موجود ولا
   انحذف هو الثاني.
3. لا تحذف أو تلمس `api.smart-link.ly` الجديد إلا بعد ما تتأكد
   الاستعادة نجحت — خليه احتياط مؤقت.

### المرحلة 2 — دمج، مو استبدال ثاني

الهدف الحقيقي اللي كان الوكيل يحاول يوصله (مطابقة تصميم
menu.smart-link.ly) **هدف صحيح** — بس نفّذه بطريقة مدمّرة. الحل:

1. اجعل `recovery-9b10d782` هو الأساس الرسمي (main الجديد)
2. من كوميتات "Smart Menu brand" التالفة (`91f5f480`, `132506d2`)
   استخرج بس العناصر البصرية المفيدة (ألوان، خطوط، تخطيط) يدويًا
   عبر `git show`، وطبّقها فوق الصفحات الموجودة أصلاً بـ Next.js —
   ما تستورد الكود كامل ولا تستبدل الصفحات
3. بخصوص ملفات smart-menu المحلية عندك (`/home/ahmed/Downloads/
   smart-menu`) — أنا ما أقدر أوصلها من بيئتي، هذا مسار على جهازك.
   الوكيل (يشتغل على جهازك) يقدر يقرأها مباشرة ويستخدمها كمرجع
   بصري فقط (ألوان/خطوط/مكونات) — بدون نسخ بنية مشروع كاملة فوق
   SmartBot.

### المرحلة 3 — تنظيم وتحصين ضد تكرار هذا

راجع أدناه البرومبت — فيه قسم "قيود صارمة" جديد كليًا يمنع أي
عملية مدمّرة مستقبلًا (force-push، حذف مشروع Vercel، git reset
لتاريخ كامل) بدون موافقة صريحة منفصلة لكل واحدة.

1. شغّل: git reflog --all | grep 9b10d782
